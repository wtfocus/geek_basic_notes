## 31 | 深度和广度优先搜索：如何找出社交网络中的三度好友关系？

1.  开篇题

### “搜索”算法

1.  算法作用于具体的数据结构之上
  
    -   **深度优先**搜索算法和**广度优先**搜索算法都是基于“**图**”这种数据结构。
2.  使用**邻接表**来存储无向图，代码如下

    -   ```java
        public class Graph { // 无向图
          private int v; // 顶点的个数
          private LinkedList<Integer> adj[]; // 邻接表
        
          public Graph(int v) {
            this.v = v;
            adj = new LinkedList[v];
            for (int i=0; i<v; ++i) {
              adj[i] = new LinkedList<>();
            }
          }
        
          public void addEdge(int s, int t) { // 无向图一条边存两次
            adj[s].add(t);
            adj[t].add(s);
          }
         }
        
        ```
    

### 广度优先搜索（BFS）

1.  原理图

    -   ![img](imgs/002e9e54fb0d4dbf5462226d946fa1ea-7355404.jpg)

2.  代码实现

    -   需求：求得s 到 t 点的最短路径

    -   ```java
        public void bfs(int s, int t) {
          // 终点和起始是一个节点
          if (s == t) return;
          // 记录已经被访问的顶点
          boolean[] visited = new boolean[v];
            // s 默认被访问
          visited[s]=true;
            // 用来存储已经被访问、但相连的顶点还没有被访问的顶点。
          Queue<Integer> queue = new LinkedList<>();
            // 默认添加起始 s 顶点到待访问队列
          queue.add(s);
            // 初始化搜索路径数组
          int[] prev = new int[v];
          for (int i = 0; i < v; ++i) {
            prev[i] = -1;
          }
            // 判断是否还存在待访问顶点
          while (queue.size() != 0) {
              // 获取一个顶点
            int w = queue.poll();
              // 循环这个顶点的所有连接顶点
           for (int i = 0; i < adj[w].size(); ++i) {
              int q = adj[w].get(i);
               // 判断这个顶点是否未被访问
              if (!visited[q]) {
                  // 添加搜索路径
                prev[q] = w;
                  // 找到终点
                if (q == t) {
                  print(prev, s, t);
                  return;
                }
                  // 标记被访问的顶点
                visited[q] = true;
                  // 添加顶点到待访问队列
            	queue.add(q);
                  }
            }
              }
         }

         private void print(int[] prev, int s, int t) { // 递归打印 s->t 的路径
              if (prev[t] != -1 && t != s) {
                    print(prev, s, prev[t]);
              }
                  System.out.print(t + " ");
              }
		 }
        ```

3.  变量说明

    -   **visited**
        -   用来记录已经被访问的顶点，用来避免顶点被重复访问。
        -   如果顶点 q 被访问，那相应的 visited[q] 会被设置为 true。
    -   **queue**
        -   一个队列，用来存储已经被访问、但是相连的顶点还没有被访问的顶点。
    -   **pre**

        -   用来记录搜索路径。当我们从顶点 s 开始，广度优先搜索到顶点 t 后，prev 数组中存储的就是搜索的路径。
        -   这个路径是反向存储的。prev[w] 存储的是，顶点 w 是从哪个前驱顶点遍历过来的。

5.  时间复杂度

    -   广度优先搜索的时间复杂度是 O(V+E)
    -   V 表示顶点的个数，E 表示连的个数。
    -   图中的所有顶点的个数都是连通的，E 肯定要大于等于 V-1，所以，广度优先搜索的时间复杂度也可以简写为 O(E)。

6.  空间复杂度

    -   空间消耗主要在几个辅助变量 visited、queue、prev 上。
    -   这个几存储空间的大小都不会超过顶点的个数，所以，空间复杂度是 O(V)。

### 深度优先搜索（DFS）

1.  原理图，从顶点 s 到顶点 t

    -   ![img](imgs/8778201ce6ff7037c0b3f26b83efba85-7357467.jpg)

2.  代码实现

    -   ```java
        boolean found = false; // 全局变量或者类成员变量
        
        public void dfs(int s, int t) {
            // 初始化变量
          found = false;
          boolean[] visited = new boolean[v];
          int[] prev = new int[v];
          for (int i = 0; i < v; ++i) {
            prev[i] = -1;
          }
            // 递归 dfs
          recurDfs(s, t, visited, prev);
          print(prev, s, t);
        }
        
        private void recurDfs(int w, int t, boolean[] visited, int[] prev) {
            // 找到，直接返回
          if (found == true) return;
            // 记录 w 顶点被访问
          visited[w] = true;
            // 如果找到终点，返回
          if (w == t) {
            found = true;
            return;
          }
            // 循环顶点所有的连接点
          for (int i = 0; i < adj[w].size(); ++i) {
            int q = adj[w].get(i);
              // 判断连接点没有被访问
            if (!visited[q]) {
                // 记录搜索路径
              prev[q] = w;
                // 递归查找子顶点
              recurDfs(q, t, visited, prev);
            }
          }
        }
        
        ```

3.  时间复杂度
  
    -   时间复杂度是 O(E)，E 表示边的个数。
4.  空间复杂度
    -   visited、prev 数据的大小跟顶点的个数 V 成正比，递归调用栈的最大深度不会超过顶点的个数
    -   所以，总的空间复杂度就是 O(V)。
